# 企业官网需求文档

## 1. 项目概述

### 1.1 项目背景
为提升企业品牌形象、拓展市场渠道、增强客户信任度，需建设一套专业的企业官方网站，作为企业对外展示的核心窗口。

### 1.2 项目目标
- 建立企业线上品牌形象，提升知名度
- 提供产品/服务展示平台，促进业务转化
- 增强客户沟通渠道，提升服务体验
- 满足移动端访问需求，覆盖多端用户

### 1.3 目标用户
| 用户类型 | 核心诉求 |
|---------|---------|
| 潜在客户 | 了解企业实力、产品服务、寻求合作 |
| 现有客户 | 获取售后支持、查看产品更新 |
| 求职者 | 了解企业文化、查看招聘信息 |
| 合作伙伴 | 了解企业资质、业务范围 |
| 投资人/媒体 | 了解企业发展、新闻动态 |

---

## 2. 技术方案

### 2.1 技术选型
| 项目 | 方案 | 说明 |
|-----|------|------|
| 页面结构 | HTML5 | 语义化标签，提升SEO与可访问性 |
| 页面样式 | CSS3 | Flexbox/Grid布局，响应式设计 |
| 交互效果 | JavaScript (原生) | 轻量交互，无需引入框架 |
| 图标 | SVG / Icon Font | 兼容性好，支持缩放 |
| 图片格式 | WebP + JPG/PNG fallback | 兼顾性能与兼容性 |
| 字体 | 系统字体栈 + 1款Web字体 | 减少加载耗时 |

### 2.2 响应式断点
| 断点 | 宽度 | 适配设备 |
|-----|------|---------|
| xs | < 768px | 手机竖屏 |
| sm | 768px - 1024px | 手机横屏 / 小平板 |
| md | 1024px - 1280px | 平板 / 小笔记本 |
| lg | ≥ 1280px | 桌面端 |

### 2.3 性能要求
- 首屏加载时间 ≤ 3秒（4G网络）
- Lighthouse 性能评分 ≥ 80
- 图片懒加载，关键资源预加载
- CSS/JS 文件压缩合并

## 3. 页面结构

### 3.1 页面清单

| 页面 | 路径 | 优先级 | 说明 |
|-----|------|--------|------|
| 首页 | index.html | P0 | 企业形象总览，核心入口 |
| 关于我们 | about.html | P0 | 企业介绍、文化、资质 |
| 产品/服务 | products.html | P0 | 产品/服务展示与详情 |
| 新闻动态 | news.html | P1 | 企业新闻、行业资讯 |
| 联系我们 | contact.html | P0 | 联系方式、在线留言 |
| 招聘页 | careers.html | P2 | 招聘信息、企业文化 |

### 3.2 全局组件

#### 3.2.1 顶部导航栏
- **位置**：页面顶部固定（sticky）
- **内容**：Logo + 导航菜单 + 语言切换（可选）+ 搜索图标
- **交互**：
  - 桌面端：横向菜单，hover高亮
  - 移动端：汉堡菜单，点击展开侧滑导航
  - 当前页面菜单项高亮标识
- **规格**：高度 64px，背景白色/半透明，滚动后加深

#### 3.2.2 页脚
- **内容**：
  - 企业简介（2-3行）
  - 快速导航链接
  - 联系方式（电话、邮箱、地址）
  - 社交媒体图标链接
  - 版权声明
- **规格**：深色背景，浅色文字，4列布局（移动端堆叠）

## 4. 页面详细需求

### 4.1 首页 (index.html)

**页面目标**：第一印象建立，引导用户深入了解

| 模块 | 内容 | 交互/规格 |
|-----|------|----------|
| Hero Banner | 企业标语 + 背景图/视频 + CTA按钮 | 全屏高度，文字居中，按钮hover动效 |
| 核心优势 | 3-4个优势卡片（图标+标题+描述） | 网格布局，hover上浮阴影 |
| 产品/服务概览 | 主推产品/服务缩略展示 | 卡片式，点击跳转详情页 |
| 数据亮点 | 关键数据展示（客户数、项目数、年限等） | 数字滚动动画，2-3行网格 |
| 客户案例/合作伙伴 | Logo墙或案例缩略 | 横向滚动或网格排列 |
| 新闻速递 | 最新2-3条新闻标题+日期 | 列表形式，点击跳转新闻页 |
| CTA区域 | "联系我们"/"获取方案"引导 | 醒目背景色，按钮居中 |

### 4.2 关于我们 (about.html)

**页面目标**：建立信任，展示企业实力与文化

| 模块 | 内容 | 交互/规格 |
|-----|------|----------|
| 企业简介 | 成立时间、规模、定位、愿景 | 左文右图布局 |
| 发展历程 | 关键里程碑时间线 | 纵向时间线，年份节点高亮 |
| 企业文化 | 使命/愿景/价值观 | 3列卡片，图标+文字 |
| 资质荣誉 | 证书/奖项图片+说明 | 网格展示，hover放大 |
| 团队风采 | 核心团队/员工活动照片 | 图片网格，hover显示说明 |

### 4.3 产品/服务 (products.html)

**页面目标**：清晰展示产品/服务，促进转化

| 模块 | 内容 | 交互/规格 |
|-----|------|----------|
| 分类导航 | 产品/服务分类标签 | 横向标签栏，点击筛选 |
| 产品/服务列表 | 名称+缩略图+简介+标签 | 卡片网格，hover显示更多 |
| 产品详情 | 完整描述、参数、优势、场景 | 单页展开或模态弹窗 |
| 方案对比 | 不同方案/套餐对比表 | 表格形式，高亮推荐项 |
| 客户案例 | 应用案例+客户评价 | 图文混排，评价引用样式 |

### 4.4 新闻动态 (news.html)

**页面目标**：展示企业活跃度，提供行业信息

| 模块 | 内容 | 交互/规格 |
|-----|------|----------|
| 分类筛选 | 企业新闻 / 行业资讯 / 活动公告 | 标签切换 |
| 新闻列表 | 标题+摘要+日期+缩略图 | 列表/卡片，支持分页 |
| 新闻详情 | 标题+正文+图片+日期+来源 | 单页展开，底部相关推荐 |
| 搜索 | 关键词搜索新闻 | 搜索框+结果列表 |

### 4.5 联系我们 (contact.html)

**页面目标**：提供沟通渠道，收集客户需求

| 模块 | 内容 | 交互/规格 |
|-----|------|----------|
| 联系信息 | 电话、邮箱、地址、工作时间 | 图标+文字，2列布局 |
| 地图 | 企业位置地图 | 嵌入第三方地图（高德/百度） |
| 在线留言 | 姓名、电话、邮箱、公司、需求描述 | 表单验证+提交反馈 |
| 社交媒体 | 微信/微博/抖音等二维码或链接 | 图标+二维码弹窗 |

## 5. 设计规范

### 5.1 色彩体系
| 类别 | 色值 | 用途 |
|-----|------|------|
| 主色 | 待定（建议深蓝/科技蓝） | 导航、按钮、标题 |
| 辅色 | 主色衍生浅色 | 背景、卡片、标签 |
| 强调色 | 橙色/金色 | CTA按钮、重要提示 |
| 中性色 | #333 / #666 / #999 / #ccc | 正文、辅助文字、边框 |
| 背景色 | #fff / #f5f7fa | 页面背景、模块背景 |

### 5.2 字体规范
| 用途 | 字号 | 字重 | 行高 |
|-----|------|------|------|
| 大标题(H1) | 36px-48px | Bold(700) | 1.2 |
| 章节标题(H2) | 28px-32px | SemiBold(600) | 1.3 |
| 模块标题(H3) | 20px-24px | Medium(500) | 1.4 |
| 正文 | 16px | Regular(400) | 1.6 |
| 辅助文字 | 14px | Regular(400) | 1.5 |
| 小字/标签 | 12px | Regular(400) | 1.4 |

### 5.3 间距规范
- 模块间距：60px-80px（桌面端），40px-60px（移动端）
- 卡片内边距：24px-32px
- 元素间距：8px / 16px / 24px / 32px（8px基准网格）

### 5.4 交互规范
| 交互 | 规格 |
|-----|------|
| 按钮 | 最小高度 40px，圆角 4px-8px，hover变色+微位移 |
| 链接 | hover下划线+变色，focus可见轮廓 |
| 卡片 | hover上浮2px+阴影加深 |
| 页面切换 | 无动画，直接切换（静态站点） |
| 滚动 | 模块渐入动画（fade-in + slide-up） |

## 6. SEO与可访问性

### 6.1 SEO要求
- 每页独立 `<title>` 和 `<meta description>`
- 语义化 HTML5 标签（header/nav/main/article/footer）
- 图片 `<alt>` 属性完整填写
- 结构化数据（JSON-LD）：Organization、Product、BreadcrumbList
- sitemap.xml 生成
- Open Graph / Twitter Card 元标签

### 6.2 可访问性
- WCAG 2.1 AA 级别合规
- 键盘可导航，focus状态可见
- 色彩对比度 ≥ 4.5:1（正文），≥ 3:1（大文字）
- 表单字段关联 `<label>`
- 跳过导航链接（Skip to content）
